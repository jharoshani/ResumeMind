## 1. Authentication Routes

These routes manage the Google OAuth 2.0 login flow. They handle redirects and do **not** return JSON.

---

### 1.1 `GET /login`

**Description:** Initiates the Google OAuth2 login flow by redirecting the browser to Google's consent screen.

**Auth Required:** ❌ No

**Request Parameters:** None

**Response:**

| HTTP Code | Behavior |
|---|---|
| `302 Found` | Redirects to `https://accounts.google.com/o/oauth2/auth` with query params: `client_id`, `redirect_uri`, `response_type=code`, `scope=openid email profile`, `state=<csrf_token>` |

**Example:**
```
GET /login HTTP/1.1
Host: localhost:5000

HTTP/1.1 302 Found
Location: https://accounts.google.com/o/oauth2/auth
  ?client_id=xxxx.apps.googleusercontent.com
  &redirect_uri=http://localhost:5000/auth/callback
  &response_type=code
  &scope=openid+email+profile
  &state=abc123random
```

**Notes:**
- The `state` parameter is a CSRF token generated by Authlib to prevent forgery attacks.
- If the user is already logged in, this still redirects to Google (allowing account switching).

---

### 1.2 `GET /auth/callback`

**Description:** OAuth2 callback endpoint. Called by Google after the user grants permission. Exchanges the authorization code for an access token, fetches the user profile, creates/updates the user in MongoDB, sets the Flask session, and redirects to the dashboard.

**Auth Required:** ❌ No (Google provides the auth code)

**Request — Query Parameters (set by Google):**

| Param | Type | Required | Description |
|---|---|---|---|
| `code` | string | ✅ Yes | Authorization code from Google |
| `state` | string | ✅ Yes | CSRF state token (must match the one generated in `/login`) |

**Responses:**

| HTTP Code | Condition | Behavior |
|---|---|---|
| `302 Found` | Success | Redirects to `/dashboard` |
| `302 Found` | Auth code exchange fails | Redirects to `/?error=auth_failed` |
| `302 Found` | Google returns error param | Redirects to `/?error=access_denied` |
| `302 Found` | User profile fetch fails | Redirects to `/?error=profile_fetch_failed` |

**Success Side Effects:**
```
1. Exchanges 'code' for access_token via POST to Google token endpoint
2. Fetches user profile from Google userinfo endpoint
3. Calls db_service.upsert_user() — creates or updates user in MongoDB
4. Sets Flask session:
     session['user_id']    = "118290764523847612345"   (Google sub)
     session['user_name']  = "Roshani Devi"
     session['user_email'] = "roshani@gmail.com"
     session['user_pic']   = "https://lh3.googleusercontent.com/..."
     session['logged_in']  = True
```

**Example Flow:**
```
1. Google redirects user to:
   GET /auth/callback?code=4/0AQlEd8x...&state=abc123random HTTP/1.1

2. Flask exchanges code → receives access_token
3. Flask fetches profile → { sub, name, email, picture }
4. Flask sets session → redirects:

   HTTP/1.1 302 Found
   Location: /dashboard
   Set-Cookie: session=<encrypted_session_cookie>; Path=/; HttpOnly
```

---

### 1.3 `GET /logout`

**Description:** Logs the user out by clearing the entire Flask session and redirecting to the landing page.

**Auth Required:** ✅ Yes (must have an active session — but gracefully handles missing session)

**Request Parameters:** None

**Response:**

| HTTP Code | Behavior |
|---|---|
| `302 Found` | Always redirects to `/` (landing page) |

**Side Effects:**
```
session.clear()  ← removes user_id, user_name, user_email, user_pic, logged_in
```

**Example:**
```
GET /logout HTTP/1.1
Host: localhost:5000
Cookie: session=<encrypted_session_cookie>

HTTP/1.1 302 Found
Location: /
Set-Cookie: session=; Expires=Thu, 01 Jan 1970; Path=/; HttpOnly
```

**Notes:**
- This does NOT revoke the Google OAuth token. The user remains logged into Google.
- If called when not logged in, still redirects to `/` (no error).

## 2. Page Routes

These routes serve **HTML pages** via Flask's Jinja2 template engine. They do **not** return JSON.

---

### 2.1 `GET /`

**Description:** Serves the landing/login page. If the user is already logged in, redirects to the dashboard.

**Auth Required:** ❌ No

**Request — Optional Query Parameters:**

| Param | Type | Description |
|---|---|---|
| `error` | string | Error code from failed auth (e.g., `auth_failed`, `access_denied`) |

**Responses:**

| HTTP Code | Condition | Behavior |
|---|---|---|
| `200 OK` | Not logged in | Renders `login.html` template |
| `302 Found` | Already logged in | Redirects to `/dashboard` |

**Template Context (when 200):**
```python
{
  "error": "auth_failed"   # or None if no error
}
```

**Page Content:**
- App branding/logo
- Brief description of the app
- "Login with Google" button → links to `/login`
- Error message banner (if `error` query param is present)

---

### 2.2 `GET /dashboard`

**Description:** Serves the main analysis dashboard page. This is the core page where users input job descriptions and upload resumes.

**Auth Required:** ✅ Yes

**Responses:**

| HTTP Code | Condition | Behavior |
|---|---|---|
| `200 OK` | Logged in | Renders `dashboard.html` template |
| `302 Found` | Not logged in | Redirects to `/` |

**Template Context (when 200):**
```python
{
  "user_name":  "Roshani Devi",
  "user_email": "roshani@gmail.com",
  "user_pic":   "https://lh3.googleusercontent.com/..."
}
```

**Page Content:**
- Navigation bar with user avatar, name, and logout button
- Job Description textarea with live word counter (max 200 words)
- PDF file upload zone (drag-and-drop, max 10 files)
- "Analyze" submit button
- Results section (initially hidden, populated by JS after analysis)
  - Ranked candidate cards with progress bars and skill badges

---

### 2.3 `GET /history` (Page)

**Description:** Serves the analysis history page. The actual history data is loaded dynamically via JavaScript calling `GET /api/history`.

**Auth Required:** ✅ Yes

**Responses:**

| HTTP Code | Condition | Behavior |
|---|---|---|
| `200 OK` | Logged in | Renders `history.html` template |
| `302 Found` | Not logged in | Redirects to `/` |

**Template Context (when 200):**
```python
{
  "user_name": "Roshani Devi",
  "user_pic":  "https://lh3.googleusercontent.com/..."
}
```

**Page Content:**
- Navigation bar (same as dashboard)
- History cards container (populated by `history.js` via `fetch('/api/history')`)
- Each card shows: date, JD snippet, number of resumes, top candidate summary
- Click a card → expands to show full results
- Delete button per card

## 3. API Routes

These routes accept and return **JSON**. All require an active Flask session.  
All are prefixed under `/api`.

---

### 3.1 `POST /api/analyze`

**Description:** The **core endpoint**. Accepts a job description and up to 10 PDF resumes, runs the AI matching engine, saves results to MongoDB, and returns ranked scores with matched skills.

**Auth Required:** ✅ Yes

**Content-Type:** `multipart/form-data`

**Request Body (FormData):**

| Field | Type | Required | Constraints |
|---|---|---|---|
| `job_description` | string | ✅ Yes | Must not be empty. Max 200 words. |
| `resumes` | file[] | ✅ Yes | Min 1 file. Max 10 files. All must be `.pdf`. |

**Frontend JavaScript Example:**
```javascript
const formData = new FormData();
formData.append('job_description', jobDescText);

// Append each selected file
for (const file of selectedFiles) {
    formData.append('resumes', file);
}

const response = await fetch('/api/analyze', {
    method: 'POST',
    body: formData
    // Note: Do NOT set Content-Type header — browser sets it with boundary
});

const data = await response.json();
```

**Success Response — `200 OK`:**
```json
{
  "status": "success",
  "analysis_id": "669f3c2a1b4e5d0012abc123",
  "total_processed": 3,
  "results": [
    {
      "rank": 1,
      "filename": "roshani_resume.pdf",
      "match_percentage": 91.5,
      "matched_skills": ["python", "flask", "mongodb", "rest", "api", "authlib"]
    },
    {
      "rank": 2,
      "filename": "aman_resume.pdf",
      "match_percentage": 84.2,
      "matched_skills": ["python", "flask", "mongodb", "rest", "api"]
    },
    {
      "rank": 3,
      "filename": "priya_resume.pdf",
      "match_percentage": 61.0,
      "matched_skills": ["python", "mongodb"]
    }
  ]
}
```

**Error Responses:**

| HTTP Code | `error` Value | `message` | Cause |
|---|---|---|---|
| `400` | `"empty_job_description"` | `"Job description cannot be empty."` | JD field is blank or whitespace |
| `400` | `"job_description_too_long"` | `"Job description exceeds 200 word limit (245 words)."` | JD has more than 200 words |
| `400` | `"no_resumes_provided"` | `"Please upload at least one PDF resume."` | No files attached |
| `400` | `"too_many_files"` | `"Maximum 10 resume files allowed. You uploaded 14."` | More than 10 files |
| `400` | `"invalid_file_type"` | `"Invalid file: photo.png. Only PDF files are accepted."` | A non-PDF file found |
| `401` | `"unauthorized"` | `"You must be logged in to perform analysis."` | No active session |
| `500` | `"processing_error"` | `"An error occurred while processing resumes."` | pdfplumber or scikit-learn crash |

**Error Response Format:**
```json
{
  "status": "error",
  "error": "too_many_files",
  "message": "Maximum 10 resume files allowed. You uploaded 14."
}
```

**Processing Pipeline:**
```
1. Validate session → 401 if missing
2. Validate JD → 400 if empty or > 200 words
3. Validate files → 400 if count > 10 or non-PDF found
4. For each PDF:
   a. Extract text via pdfplumber
   b. Clean text via regex (lowercase, strip emails/phones/urls)
5. Clean JD text
6. For each resume text:
   a. TfidfVectorizer → transform both JD and resume
   b. cosine_similarity → match_percentage
   c. Extract non-zero overlap indices → matched_skills
7. Sort results by match_percentage DESC
8. Assign rank 1, 2, 3...
9. Save to MongoDB via db_service.save_analysis()
10. Return JSON response
```

### 3.2 `GET /api/history`

**Description:** Returns a paginated list of the logged-in user's past analysis sessions, sorted by most recent first.

**Auth Required:** ✅ Yes

**Request — Query Parameters:**

| Param | Type | Required | Default | Description |
|---|---|---|---|---|
| `limit` | int | ❌ No | `20` | Number of records per page (max 50) |
| `page` | int | ❌ No | `1` | Page number (1-indexed) |

**Frontend JavaScript Example:**
```javascript
const response = await fetch('/api/history?limit=20&page=1');
const data = await response.json();
```

**Success Response — `200 OK`:**
```json
{
  "status": "success",
  "total": 12,
  "page": 1,
  "limit": 20,
  "history": [
    {
      "analysis_id": "669f3c2a1b4e5d0012abc123",
      "created_at": "2026-07-03T14:05:00Z",
      "job_description_snippet": "We are looking for a Python backend developer with experience in...",
      "total_resumes": 3,
      "top_candidate": {
        "filename": "roshani_resume.pdf",
        "match_percentage": 91.5
      }
    },
    {
      "analysis_id": "669f2a001b4e5d0012abc456",
      "created_at": "2026-07-02T10:30:00Z",
      "job_description_snippet": "Frontend developer needed for React-based dashboard application...",
      "total_resumes": 5,
      "top_candidate": {
        "filename": "kiran_resume.pdf",
        "match_percentage": 77.8
      }
    }
  ]
}
```

**Error Responses:**

| HTTP Code | `error` Value | `message` | Cause |
|---|---|---|---|
| `401` | `"unauthorized"` | `"You must be logged in to view history."` | No active session |
| `400` | `"invalid_page"` | `"Page number must be a positive integer."` | page ≤ 0 or non-numeric |
| `400` | `"invalid_limit"` | `"Limit must be between 1 and 50."` | limit ≤ 0 or > 50 |

**Notes:**
- `total` is the total count of all analyses by this user (not just the current page).
- Each history item includes a `top_candidate` summary for quick display.
- Results in each history item are NOT included (use `GET /api/history/:id` for full detail).

### 3.3 `GET /api/history/:id`

**Description:** Returns the full detailed results of a specific past analysis, including all candidate scores and matched skills.

**Auth Required:** ✅ Yes

**Request — URL Parameters:**

| Param | Type | Required | Description |
|---|---|---|---|
| `id` | string | ✅ Yes | MongoDB ObjectId of the analysis document |

**Frontend JavaScript Example:**
```javascript
const analysisId = "669f3c2a1b4e5d0012abc123";
const response = await fetch(`/api/history/${analysisId}`);
const data = await response.json();
```

**Success Response — `200 OK`:**
```json
{
  "status": "success",
  "analysis_id": "669f3c2a1b4e5d0012abc123",
  "created_at": "2026-07-03T14:05:00Z",
  "job_description_snippet": "We are looking for a Python backend developer with experience in...",
  "total_resumes": 3,
  "results": [
    {
      "rank": 1,
      "candidate_filename": "roshani_resume.pdf",
      "match_percentage": 91.5,
      "matched_skills": ["python", "flask", "mongodb", "rest", "api", "authlib"]
    },
    {
      "rank": 2,
      "candidate_filename": "aman_resume.pdf",
      "match_percentage": 84.2,
      "matched_skills": ["python", "flask", "mongodb", "rest", "api"]
    },
    {
      "rank": 3,
      "candidate_filename": "priya_resume.pdf",
      "match_percentage": 61.0,
      "matched_skills": ["python", "mongodb"]
    }
  ]
}
```

**Error Responses:**

| HTTP Code | `error` Value | `message` | Cause |
|---|---|---|---|
| `401` | `"unauthorized"` | `"You must be logged in to view analysis details."` | No active session |
| `400` | `"invalid_id"` | `"Invalid analysis ID format."` | ID is not a valid ObjectId |
| `403` | `"forbidden"` | `"You do not have permission to view this analysis."` | Analysis belongs to another user |
| `404` | `"not_found"` | `"Analysis not found."` | No document with this ID exists |

**Security:**
- The endpoint verifies that `document.user_id == session['user_id']` before returning data.
- This prevents users from guessing ObjectIds to access other users' data.

### 3.4 `DELETE /api/history/:id`

**Description:** Deletes a specific analysis record. Only the owner can delete their own records.

**Auth Required:** ✅ Yes

**Request — URL Parameters:**

| Param | Type | Required | Description |
|---|---|---|---|
| `id` | string | ✅ Yes | MongoDB ObjectId of the analysis to delete |

**Frontend JavaScript Example:**
```javascript
const analysisId = "669f3c2a1b4e5d0012abc123";
const response = await fetch(`/api/history/${analysisId}`, {
    method: 'DELETE'
});
const data = await response.json();
```

**Success Response — `200 OK`:**
```json
{
  "status": "success",
  "message": "Analysis deleted successfully."
}
```

**Error Responses:**

| HTTP Code | `error` Value | `message` | Cause |
|---|---|---|---|
| `401` | `"unauthorized"` | `"You must be logged in to delete analyses."` | No active session |
| `400` | `"invalid_id"` | `"Invalid analysis ID format."` | ID is not a valid ObjectId |
| `403` | `"forbidden"` | `"You do not have permission to delete this analysis."` | Analysis belongs to another user |
| `404` | `"not_found"` | `"Analysis not found."` | No document with this ID exists |

**Side Effects:**
- Permanently removes the document from the `analyses` collection.
- This action is irreversible.

## 4. Error Response Format

All API error responses (from `/api/*` routes) follow this consistent structure:

```json
{
  "status": "error",
  "error": "<error_code>",
  "message": "<human_readable_message>"
}
```

### Complete Error Code Reference

| Error Code | HTTP Status | Endpoint(s) | Description |
|---|---|---|---|
| `unauthorized` | `401` | All `/api/*` routes | No active Flask session |
| `empty_job_description` | `400` | `POST /api/analyze` | JD is blank/whitespace |
| `job_description_too_long` | `400` | `POST /api/analyze` | JD exceeds 200 words |
| `no_resumes_provided` | `400` | `POST /api/analyze` | No PDF files uploaded |
| `too_many_files` | `400` | `POST /api/analyze` | More than 10 files |
| `invalid_file_type` | `400` | `POST /api/analyze` | Non-PDF file detected |
| `invalid_id` | `400` | `GET/DELETE /api/history/:id` | Malformed ObjectId string |
| `invalid_page` | `400` | `GET /api/history` | Bad pagination value |
| `invalid_limit` | `400` | `GET /api/history` | Limit out of range (1-50) |
| `forbidden` | `403` | `GET/DELETE /api/history/:id` | User doesn't own this record |
| `not_found` | `404` | `GET/DELETE /api/history/:id` | No document with this ID |
| `processing_error` | `500` | `POST /api/analyze` | Internal processing failure |
